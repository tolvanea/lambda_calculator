Lambda calculator

What is this?

Untyped lambda calculus evaluation tool. It happens to also be a turing complete programming language.

Umm, what?

This is a lambda function definition 

λab.a

It takes two parameters a and b, and returns the first parameter a.

Evaluation of a lambda function is made by putting it inside the parenthesis with the arguments: 

(λab.a parameter1 parameter2)

So instead of the common function call notation f(a, b), it is (f a b). Well anyways, in this case the function returns the first argument, so it would evaluate to parameter1.

The parameter1 is some expression, which is either of the above. That is, it is either

  • function definition λ..., or
  • function evaluation (...)

For example, this is a valid program you can insert to left 

parameter1 = λc.c
parameter2 = λd.d
in
(λab.a parameter1 parameter2)
which evaluates to 
λc.c

Ok, and?

That's it! There is nothing more! No classes, no numbers, no booleans. Only pure lambda functions. The task here is to build everything from bottom up starting from lambda functions. For example, a boolean calculation "FALSE AND TRUE" can be encoded like so: 

true = λab.a
false = λab.b
and = λxy.(x y false)
in
(and true false)
Try copy-pasting it to the left, it evaluates to 
λa.λb.b

Wait why λa.λb.b instead of λab.b?

Because it's internal representation of lambda calculus, and it does not have multiparameter functions. A two parameter function λab is actually just a syntax sugar to a single parameter function λa that returns a single parameter function λb. You can feed two parameters in and it works as if it was a two parameter function.

But whyyyyyy?

You can formulate every computation on functions! No values required! It's like nand-game but with functions instead of nand-gates. I was inspired by this video, see it, it's awesome:

https://www.youtube.com/watch?v=RcVA8Nj6HEo

Standard library

Copy paste whatever fragments you need from here. 

true = λab.a
false = λab.b
and = λxy.(x y false)

# Numbers encoded as recursive function calls
0 = λfx.x
1 = λfx.(f x)
2 = λfx.(f (f x))
3 = λfx.(f (f (f x)))
4 = λfx.(f (f (f (f x))))
+1 = λnfx.(f (n f x))
-1 = λnfx.(n λgh.(h (g f)) (λu.x) (λu.u))

# Arithmetic
+ = λnmfx.((n f) (m f x))
* = λnmf.(m (n f))
- = λmn.n -1 m
^ = λnm.(m n)
↑↑ = λmn.((-1 n) (^ m) m)

# Pairs: x=first, y= second, h=head, t=tail
pair = λxyf.(f x y)
nil = false
fst = λl.(l λhtd.h nil)
snd = λl.(l λhtd.t nil)
is_nill = λl.(l λhtd.false true)

# Recursion with Y-combinator
u = λxy.(y (x x y))
Y = (u u)

# Factorial, e.g. (! 3) = 3*2*1 = 6
if_0 = λn.((n λa.false) true)
factorial = λfn.(if_0 n 1 (* n (f (-1 n))))
! = (u u factorial)

# List processing, e.g.
# (rfold + 0 (map (^ 2) [1,2,3])) = 1+4+9
rfold = λfa.(Y (λrl.(l λhtd.(f (r t) h) a)))
map = λf.(rfold (λah.(pair (f h) a)) nil)
[1,2,3] = (pair 1 (pair 2 (pair 3 nil)))
